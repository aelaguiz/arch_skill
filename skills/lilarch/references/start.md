# Lilarch Start Mode

## Goal

Create or repair the compact doc so planning can happen safely.

## Write in start mode

- TL;DR
- North Star
- `lilarch:block:requirements`
- explicit defaults
- explicit non-requirements
- the small set of clarifying questions that truly block planning

## Clarifying-question bar

Ask only when the answer is both:

- not discoverable from repo evidence
- required to avoid planning the wrong thing

Good examples:

- a UX default that is nowhere in the repo
- whether an old behavior must be preserved for rollout

Bad examples:

- asking where the code lives before searching
- asking whether tests exist before reading the test tree

## Stop condition

- If a required product or UX answer is missing, stop for that answer.
- Otherwise finish with a doc that is ready for lilarch plan mode.
